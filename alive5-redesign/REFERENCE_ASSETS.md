## How to use these references

**Option A (recommended for offline reliability):**
- Use the local copies in `references/twine/images/` and `references/todesktop/images/`
- These are bundled with the project

**Option B (always-fresh, requires internet):**
- Reference the original URLs above in image tags
- Drawback: URLs may break if Twine/ToDesktop redeploy

**Option C (best for production):**
- Use these references for VISUAL inspiration only
- Replace with custom Alive5-themed illustrations during the build
- The Alive5 CDN URLs in `audit/all-images.json` should be the actual production assets
